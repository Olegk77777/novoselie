## Что мы делаем
Браузерная казуальная игра: обустройство квартиры в эстетике пост-советских 90-х. Полный концепт — в файле CONCEPT.md (прочитай его первым делом). Текущая цель — версия v0.1 по плану из концепта.

## Технологии (НЕ менять без обсуждения)
- Чистый HTML + JavaScript + Three.js (подключаем через CDN, без сборщиков/npm на старте)
- Один запуск: открыть index.html через локальный сервер (`python3 -m http.server` — объясни Олегу как)
- Изометрическая камера, зафиксирована, поворот только шагами по 90°
- Сетка на полу, предметы ставятся по клеткам
- Низкополигональные модели собираем КОДОМ из примитивов Three.js (BoxGeometry и т.п.) — стиль PS1
- Текстуры Олег кладёт в папку /textures (генерирует сам) — натягиваем на модели

## Архитектура (железные правила)
1. **Предметы = данные, не код.** Все предметы описаны в items.json: id, размер в клетках, очки уюта, можно ли на стену/пол, текстура. Новый предмет = новая запись в JSON, без правки логики.
2. **Локализация с первого дня.** ВСЕ тексты игры — только в locales/ru.json (потом добавим en.json). В коде — только ключи. Ни одной русской строки внутри .js файлов.
3. **Квесты = данные.** Условия квестов описываем декларативно в quests.json (тип условия + параметры), движок их проверяет.
4. **Никаких реальных брендов** — названия, логотипы, узнаваемые товары только выдуманные.

## Текстуры — как ставить задачу Олегу (железное правило)
Текстуры генерирует Олег сам. Каждый раз, когда нужна текстура, Клод даёт задание в таком формате:
- **Имя файла и путь:** например `textures/floor_parquet.png` (только латиница, нижний регистр, подчёркивания)
- **Размер:** степень двойки, по умолчанию 512×512 (большие поверхности — 1024×1024)
- **Формат:** PNG (или JPG, если без прозрачности и важен вес)
- **Бесшовность:** обязательно указывать, нужна ли seamless/tileable текстура
- **Промпт для генерации:** готовый, на английском, без параметров-флагов (--ar и т.п.)

Пока текстуры нет — модель работает с простым цветом-заглушкой, игра не ждёт.

Файлы от Олега перед коммитом приводить в порядок: проверять имя (пробелы и двоеточия из путей вылезают в имя файла), сжимать до рабочего размера (`sips -z 1024 1024 -s format jpeg -s formatOptions 80 ...`), цель — не больше ~500 КБ на текстуру. Оригиналы не удалять, а переносить в ~/Downloads.

## Технические решения (зафиксировано, не менять молча)
- **Three.js 0.170.0**, подключён через importmap в index.html (CDN unpkg). Версию не обновлять без причины.
- **Запуск игры:** из папки проекта `python3 -m http.server 8123`, потом открыть http://localhost:8123 в браузере. Просто двойной клик по index.html НЕ работает (браузер блокирует загрузку модулей и JSON с диска).
- **Система координат:** 1 клетка сетки = 1 юнит Three.js. Пол центрирован в начале координат (сетка 10×8 → x от −5 до 5, z от −4 до 4). Центр клетки (col,row): x = −cols/2 + col + 0.5, z = −rows/2 + row + 0.5. Формулы — в src/grid.js.
- **Камера:** ортографическая (изометрия), позиция (10,10,10), смотрит в (0,0,0).
- **Референс планировки:** Planirovka.png в корне — образ всей квартиры на будущее (v0.3+), в v0.1 строим одну комнату.
- **GitHub:** репозиторий https://github.com/Olegk77777/novoselie (пока публичный, перед релизом закроем). Деплой: обычный `git push` в main → GitHub Pages обновляет игру на https://olegk77777.github.io/novoselie/ за 1–2 минуты. Файл .nojekyll в корне обязателен (отключает лишнюю обработку Pages).
- **География релиза:** Олег в Украине → Яндекс.Игры НЕ предлагать и не использовать. Целевые площадки: itch.io (старт), затем CrazyGames / GameDistribution / Poki (международные HTML5). GitHub Pages — только наш тестовый стенд, не релизная площадка.
