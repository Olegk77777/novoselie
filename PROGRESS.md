### 14 июня 2026 — Туман-фон: многослойная светящаяся мгла вместо плоской синей заливки (запрос Олега)
- **Зачем:** фон был плоский `scene.background = 0x1a1a2e` — мёртвая синяя заливка. Олег попросил красивый многослойный анимированный туман с параллаксом и лёгким синим свечением, который по мере открытия комнат будет рассеиваться; язычки тумана могут заплывать на открытые (без стен) ближние кромки. Это работает на думерскую атмосферу (квартира — тёплый островок в холодной светящейся пустоте «нежилых комнат»).
- **Как сделано (src/fog.js):** два полноэкранных квада в КЛИП-ПРОСТРАНСТВЕ (PlaneGeometry 2×2, вершинный шейдер кладёт позицию напрямую — камера не нужна, туман всегда на весь экран, не зависит от зума/сдвига комнаты, `frustumCulled=false`, `depthTest/Write=false`, `toneMapped=false`):
  - **BACKDROP** (`renderOrder −1000`, непрозрачный, за комнатой): 3 параллакс-слоя дешёвого value-noise (как в walls.js) с разной скоростью/масштабом (дрейф 1:2.3:4.2 = глубина при статичной камере) + лёгкий domain-warp на дальнем слое (клубы) + воздушная перспектива. Тело тумана держим НИЖЕ порога bloom — комнату не засвечивает.
  - **Синее свечение** = 2–3 «фонарные лужи» (барвинково-синие, мягкое гало + тугое ядро). Ядро по яркости (luma ~0.8) пробивает порог bright-pass bloom → пост-эффект (src/bloom.js) сам раздувает его в мягкое синее гало («фонарь во дворе сквозь мглу»).
  - **VEIL** (`renderOrder +1000`, прозрачный, NormalBlending, поверх комнаты): редкие холодные язычки ТОЛЬКО у низа кадра (ранний выход выше ~48% — верх кадра почти бесплатный, интерьер не трогаем) — наползают на открытые ближние кромки пола.
- **Рассеивание:** один юниформ `uClear` 0..1 согласованно тонит туман (плотность ↓ + линия высоты опускается — небо чистится первым + фонари гаснут). Сейчас цель = доля шкалы уюта × 0.35 (мягкий суррогат «открытия комнат»: на 100% мгла лишь чуть отступает, думер-настроение сохраняем), плавный лерп в кадровом цикле. Полный диапазон 0..1 оставлен на v0.3+ (вторая комната). API чистый — позже привязать к числу открытых комнат.
- **Перф:** value-noise без fbm выше 3 октав; на слабом железе (ретина+сенсор ≈ iPad) настоящий препроцессорный `#define LOW_END` ВЫКИДЫВАЕТ дальний слой, domain-warp и вторые октавы (мёртвый код удаляет компилятор). Легче оконного шейдера, который и так работает.

## Идеи на потом
- **Игра со светом — отдельная механика атмосферы (от Олега, 13.06.2026):** свет = смысл игры, не только оформление. Светящиеся декор-предметы, которые игрок ставит ради настроения: **лава-лампа** (✓ СДЕЛАНО 14.06.2026 — шейдерные метаболы + иридесцентный point light, работает без розетки), **неоновые таблички** на стену (emissive-линии холодного цвета + цветной свет + мягкий bloom под флагом десктопа), **светящиеся фигурки/ночники** на стену и полки. Технически всё уже готово ложиться ОДНОЙ записью в items.json: `surface:"emissive"` + `makeApplianceLight` в builder (риг и гашение по току есть). Возможные развития: выключатель общего света (KEY) — вечерний режим «только лампы»; «оценка атмосферы» за удачные световые сочетания (тёплый+холодный контраст); смена exposure/настроения. Перф-потолок — ~4-5 PointLight без теней одновременно (лишние светящиеся предметы → чистый emissive без источника).
